# Save / Void / Cancel buttons appear in different places depending on the transaction

Status: needs-triage
Severity: minor
Area: Transaction register

When selecting a transaction for editing, the Save, Void, and Cancel buttons appear in
different places depending on the transaction's features (e.g. currencies of the legs, tags,
notes). This is confusing and annoying.

The buttons should always appear in the same place — e.g. at the "new row" position relative
to the other controls in the dock, on the right side of the dock.
