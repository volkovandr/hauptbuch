# Selecting a wrong-currency target account in a split breaks the register and locks the dock

Status: resolved
Category: bug
Severity: medium
Area: Transaction register

When entering a large multi-currency split transaction, selecting a `To -> <account>` that
points to an account in a different currency than the transaction currency is (correctly)
rejected: the target account must be in the same currency. But there are three problems:

1. The problem is only surfaced on Save. It would be better to tell the user immediately, when
   the account is selected, that it is not allowed.
2. When the problem appears, the transaction register table disappears. This is simply wrong.
3. After selecting the correct account, the Save button still does nothing — and no error
   appears in the logs. The same error message stays on screen and nothing works. Cancel does
   not work either; the only way out is to refresh the page.

### Real root cause (confirmed from the htmx 2.0.4 source)

The commit form targets `#register-body` (`hx-swap="outerHTML"`). On a validation error the server
returned the dock/panel fragment as an **out-of-band-only** response (`hx-swap-oob="true"`). htmx
strips the OOB element out of the response before the main swap (`He` → `o.parentNode.removeChild`),
leaving the **main fragment empty**, then swaps that emptiness into `#register-body` with
`outerHTML` — **deleting the table**. Problem 3 follows: with `#register-body` gone, the next Save
has no target (`htmx:targetError`), so it silently no-ops (and may have committed server-side).

Both the split panel **and** the plain dock error paths had this exact shape.

### Fix (applied, `check` green; awaiting owner browser confirmation)

Set `HX-Reswap: none` on every commit/void **error** response (`RegisterSplitController.commitError`,
`RegisterEntryController.dockError`). htmx's `_e` treats `none` as a no-op main swap while still
processing OOB — so the register is left completely untouched and stays a valid target for the next
Save. Tests updated to assert the `HX-Reswap: none` header (the real guarantee) instead of the
absence of rows, plus a cross-currency regression mirroring the owner's exact flow.
